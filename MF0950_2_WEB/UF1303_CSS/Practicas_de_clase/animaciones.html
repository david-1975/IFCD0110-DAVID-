Las animaciones CSS

Una animación permite a un elemento de cambiar gradualmente de un estilo a otro. Se puede cambiar tantas propiedades CSS que desee, como muchas veces que desee. Para utilizar la animación CSS, primero debe especificar algunos fotogramas clave para la animación.

Fotogramas clave conservar lo estilos el elemento tendrá en determinados momentos.

Ejecutar la animación en la dirección contraria o ciclos alternos

La animation-directionpropiedad especifica si una animación debe ser jugado hacia delante, hacia atrás o en ciclos alternos.

La propiedad de animación dirección puede tener los siguientes valores:

  • normal- La animación se juega como normales (hacia delante). Esto es por defecto
  • reverse - La animación se juega en dirección inversa (hacia atrás)
  • alternate - La animación se reproduce hacia delante en primer lugar, luego hacia atrás
  • alternate-reverse - La animación se reproduce hacia atrás al principio, y luego hacia delante

Especificar la curva de velocidad de la animación

La animation-timing-functionpropiedad especifica la curva de velocidad de la animación.

La propiedad-tiempo-función de animación puede tener los siguientes valores:

  • ease - Especifica una animación con un comienzo lento, luego rápido, y luego terminar lentamente (esto es por defecto)
  • linear - Especifica una animación con la misma velocidad de principio a fin
  • ease-in - Especifica una animación con un comienzo lento
  • ease-out - Especifica una animación con un extremo lenta
  • ease-in-out - Especifica una animación con un lento comienzo y fin
  • cubic-bezier(n,n,n,n) - le permite definir sus propios valores de una función cúbica-Bezier

specificar el modo de relleno para una animación

animaciones CSS no afectan a un elemento antes de que el primer fotograma clave se juega o después de que se jugó el último fotograma clave. La propiedad-fill-mode animación puede anular este comportamiento.

La animation-fill-modepropiedad especifica un estilo para el elemento de destino cuando la animación no se ejecuta (antes de que comience, después de que termine, o ambos).

La propiedad-fill-mode animación puede tener los siguientes valores:

  • none- Valor por defecto. La animación no se aplicará ningún estilo al elemento antes o después de que se esté ejecutando
  • forwards - El elemento conservará los valores de estilo que se define por el último fotograma clave (depende de la animación dirección y animación-iteración de conteo)
  • backwards - El elemento obtendrá los valores de estilo que se define por el primer fotograma clave (depende de la animación-dirección), y retener a esto durante el periodo de retardo de animación
  • both - La animación seguir las reglas para tanto hacia delante y hacia atrás, extendiendo las propiedades de animación en ambas direcciones
Property Description
@keyframes Especifica el código de animación.
animation Una propiedad abreviada para establecer todas las propiedades de animación.
animation-delay Especifica un retraso para el inicio de una animación.
animation-direction Especifica si una animación debe reproducirse hacia adelante, hacia atrás o en ciclos alternos.
animation-duration Especifica el tiempo que debe durar una animación para completar un ciclo
animation-fill-mode Especifica un estilo para el elemento cuando la animación no se está reproduciendo (antes de que comience, después de que termine, o ambas cosas)
animation-iteration-count Especifica el número de veces que se debe reproducir una animación.
animation-name Especifica el nombre de la animación @keyframes.
animation-play-state Especifica si la animación se está ejecutando o está en pausa.
animation-timing-function Especifica la curva de velocidad de la animación.
